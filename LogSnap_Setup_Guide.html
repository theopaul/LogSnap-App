LogSnap - Xcode Project Setup Guide

Note: This guide will help you set up a new Xcode project for LogSnap and incorporate the files we've already created.

Step 1: Create a New Xcode Project

Launch Xcode

Open Xcode from your Applications folder or Spotlight.

If Xcode is not installed, download it from the Mac App Store.

Create a New Project

From Xcode's welcome screen, click "Create a new Xcode project" or go to File > New > Project.

Choose the iOS App Template

Choose "App" under the iOS section and click "Next".

Configure Your Project

Enter the following details:

  • Product Name: LogSnap
  • Organization Identifier: com.yourname.logsnap (use your own domain if you have one)
  • Interface: SwiftUI
  • Life Cycle: SwiftUI App
  • Language: Swift
  • ✓ Use Core Data (ensure this is checked)
  • ✓ Include Tests (optional)

Click "Next".

Choose a Location

Select where you want to save your project, preferably in a new folder.

Make sure "Create Git repository" is checked for version control.

Click "Create".

Step 2: Integrate Existing Code

Create Project Structure

In Xcode's Project Navigator (left sidebar), create the following folder structure:

LogSnap/
├── App/
├── Core/
│   ├── Extensions/
│   ├── Utilities/
│   └── Localization/
├── Data/
│   ├── CoreData/
│   └── Repositories/
├── Features/
│   ├── Products/
│   │   ├── Models/
│   │   ├── ViewModels/
│   │   └── Views/
│   └── Suppliers/
│       ├── Models/
│       ├── ViewModels/
│       └── Views/
├── UI/
│   ├── Components/
│   └── Themes/
└── Main/

To create folders: Right-click on the parent folder > New Group > Name it accordingly.

Add Files to Project

For each Swift file we've created:

  1. Go to where you stored all the code files we created
  2. Copy the file content or the file itself
  3. In Xcode, right-click on the appropriate folder in the Project Navigator
  4. Select "New File..." and choose "Swift File"
  5. Name it exactly as the original file
  6. Paste the content if you copied it, or if you have the file, drag it into the folder

Pay Special Attention to:

  • Make sure CoreDataManager.swift is properly integrated and matches the generated CoreData model
  • Ensure all localization files are in the correct places
  • Check that the LogSnapApp.swift file properly references the MainTabView

Step 3: Configure the Project

Set Up Localization

To enable localization for English and Portuguese:

  1. Select your project in the Project Navigator
  2. Select the "Info" tab
  3. Under "Localizations", click the "+" button
  4. Add "Portuguese (Brazil)" or "Portuguese (Portugal)"
  5. Select which files to localize

Set Deployment Target

Make sure your deployment target is set to iOS 15.0 or higher.

Step 4: Build and Run

Test Your App

Click the Play button (▶) in the top-left corner of Xcode to build and run your app in the simulator.

If you encounter any build errors, check the console for details and fix them one by one.

Common Build Issues:

  • Missing files: Ensure all referenced files are added to the project
  • Duplicate declarations: Make sure each component is defined only once
  • CoreData model mismatch: Verify your CoreData model matches the entities used in the code
  • Missing imports: Check that all required frameworks are imported

Need Help?

If you encounter issues setting up the project, see the detailed documentation in the README.md file or contact support.
